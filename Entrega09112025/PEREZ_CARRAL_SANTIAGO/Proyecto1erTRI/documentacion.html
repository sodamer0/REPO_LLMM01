2. Descripción del Proyecto

2.1. Temática Elegida

Se ha desarrollado un portfolio personal para un profesional del diseño web y la programación en general. Esta temática fue seleccionada porque permite demostrar de manera práctica todos los conceptos aprendidos en HTML5 y CSS3, incluyendo estructura semántica y creación de formularios.

2.2. Justificación

Un portfolio es una herramienta fundamental para cualquier profesional del sector tecnológico. Permite mostrar habilidades, proyectos y experiencia de forma visual y accesible. Además, este tipo de sitio web requiere implementar diversos elementos que demuestran el manejo de HTML5 Y CSS3.

2.3. Objetivos Específicos

  • Crear una estructura HTML5 semántica y accesible
  • Implementar un diseño responsive que funcione en diferentes dispositivos
  • Aplicar el modelo de caja de CSS de forma efectiva
  • Desarrollar un formulario de contacto funcional y completo
  • Utilizar layouts modernos con Flexbox y Grid
  • Mantener código limpio, comentado y bien organizado

2.4. Público Objetivo

El sitio está dirigido a:

  • Potenciales clientes que buscan servicios de diseño web
  • Empresas que necesitan contratar desarrolladores frontend
  • Empresas que necesitan contatar programadores backend
  • Reclutadores técnicos del sector IT

5. Dificultades y Soluciones

5.1. Dificultad: Diseño Responsive

Problema: Inicialmente, el diseño se rompía en pantallas pequeñas, especialmente la navegación y las tarjetas con Grid.

Solución: Implementé media queries en CSS para adaptar el layout en dispositivos móviles. Cambié la navegación de horizontal a vertical y los grids de 3 columnas a 1 columna en pantallas menores de 768px.

@media (max-width: 768px) {
  .grid-habilidades {
    grid-template-columns: 1fr;
  }
}

5.2. Dificultad: Centrado Vertical con Flexbox

Problema: Tuve problemas para centrar verticalmente elementos dentro de contenedores flexibles.

Solución: Aprendí a combinar las propiedades justify-content y align-items correctamente. justify-content alinea en el eje principal y align-items en el eje cruzado.

5.3. Dificultad: Validación de Formularios

Problema: Los atributos de validación HTML5 no funcionaban en todos los campos.

Solución: Investigué en MDN Web Docs y añadí atributos específicos como pattern para validación de teléfonos, minlength/maxlength para textos, y required en campos obligatorios.

5.4. Dificultad: Organización del Código CSS

Problema: El archivo CSS se volvió muy extenso.

Solución: Organicé el código en secciones comentadas (Reset, Tipografía, Navegación, etc.) siguiendo una metodología lógica. Esto facilitó encontrar y modificar estilos específicos.
